1. Description

Le composant maj_github automatise la publication en 4 étapes :

  1. Synchronisation des .docx modifiés (sync_dossiers.py)
  2. Génération du site (lancer.cmd nolocal)
  3. Vérification du token GitHub (API)
  4. Commit et push vers GitHub
⚠️ Pare-feu Windows — prérequis
Windows peut bloquer git-credential-manager.exe lors du premier push.
Autoriser ce programme en trafic sortant dans le Pare-feu Windows Defender avant la première utilisation.
Chemin typique : C:\Program Files\Git\mingw64\bin\git-credential-manager.exe

4. Authentification GitHub (v1.15+)

Priorité 1 — Token dans config.yaml (recommandé)

Token valide → push silencieux à chaque fois, aucune fenêtre.
  1. github.com → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Cocher tout le bloc repo
  3. Expiration recommandée : No expiration
  4. Copier le token dans prog\config.yaml

Priorité 2 — GCM (GitHub Desktop, sans token)

Laisser github_token vide → GCM prend la main. Nécessite que le pare-feu autorise git-credential-manager.exe.
