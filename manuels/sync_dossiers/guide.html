Guide — sync_dossiers
Synchronisation de dossiers avec sélection visuelle
Version1.0
DateMars 2026
Dossiermanuel\sync_dossiers\

1. Description

sync_dossiers.py synchronise un dossier source (vos .docx en cours d'édition) vers un dossier cible (le dépôt git). Seuls les fichiers plus récents dans la source sont copiés.

2. Utilisation

Via MAJ_GITHUB.cmd (automatique)

sync_dossiers est appelé automatiquement en étape 1 de maj_github.py. Aucune action manuelle nécessaire.

Utilisation standalone

python prog\sync_dossiers.py prog\config.yaml

Ou drag & drop d'un fichier .yaml sur l'icône du programme.

3. Configuration (dans config.yaml)

ChampDescription
racine_sourceDossier de travail contenant les .docx modifiés
racine_site_localDossier cible (dépôt git local)
filtreExpression régulière pour filtrer les fichiers (vide = tout accepter)

Exemple de filtre

filtre = .*\.docx$,.*\.png$

Copie uniquement les .docx et .png.

4. Interface de sélection

Une fenêtre Tkinter affiche les fichiers à synchroniser. L'utilisateur peut décocher des fichiers avant de confirmer.

5. Comportement

  • Seuls les fichiers plus récents dans la source sont proposés à la copie
  • Aucun fichier n'est supprimé dans la cible
  • Le message de résultat ne s'affiche qu'en cas d'erreur (comportement silencieux en succès)
