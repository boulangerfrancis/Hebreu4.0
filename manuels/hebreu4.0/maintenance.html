1. Installation initiale (nouvelle machine)

Prerequis : Python 3.11+, Node.js, PDFCreator, GitHub Desktop, acces internet.
Pare-feu Windows — etape obligatoire
Windows bloque git-credential-manager.exe par defaut, ce qui empeche tout push GitHub.
Avant le premier push :
1. Ouvrir Pare-feu Windows Defender → Parametres avances → Regles de trafic sortant
2. Nouvelle regle → Programme → parcourir jusqu'a git-credential-manager.exe
3. Chemin typique : C:\Program Files\Git\mingw64\bin\git-credential-manager.exe
4. Autoriser la connexion
Sans cette autorisation : "Impossible de se connecter au serveur distant (403)"
  1. Installer GitHub Desktop et se connecter a son compte GitHub
  2. Cloner le depot via GitHub Desktop : https://github.com/GuyTitt/Hebreu4.0
  3. Dezipper le package dans C:\SiteGITHUB\Hebreu4.0\
  4. python package\prog\installer.py — cree C:\virpy13 + bibliotheques
  5. Editer prog\config.yaml (voir section 2)
  6. Autoriser git-credential-manager.exe dans le pare-feu (voir encadre ci-dessus)
  7. python package\prog\remplace.py — deploie programmes ET manuels
  8. python prog\versions.py — verifie l'installation
  9. lancer.cmd — premier test

2. Configuration (prog\config.yaml)

Ce fichier est personnel. Ne jamais le commiter sur git.
Laisser github_token vide si vous utilisez GCM (GitHub Desktop).
racine_source     = C:\MonDossier\documents
racine_site_local = C:\SiteGITHUB\Hebreu4.0
lancer_cmd        = C:\SiteGITHUB\Hebreu4.0\prog\lancer.cmd
url_github        = https://github.com/GuyTitt/Hebreu4.0
github_token      =           # laisser vide = GCM (GitHub Desktop)
github_user       =           # laisser vide = GCM (GitHub Desktop)
branche           = main
message_commit    = auto
confirmation_git  = true

Token PAT : github.com → Settings → Developer settings → Personal access tokens → Tokens (classic) → cocher repo.

3. Mise a jour des modules

3.1 Modifier un programme

  1. Editer package\prog\genere_site_v25.6.py
  2. Renommer en genere_site_v25.7.py
  3. Mettre a jour version = ("genere_site.py", "25.7")
  4. Ajouter # v25.7 : description en tete
  5. python package\prog\remplace.py

3.2 Modifier un manuel

  1. Editer package\manuels\hebreu4.0\maintenance_v1.2.html
  2. Renommer en maintenance_v1.3.html
  3. Mettre a jour la meta version : <meta name="version" content="1.3">
  4. python package\prog\remplace.py → deploie vers manuels\hebreu4.0\maintenance.html

3.3 Archiver un ancien fichier

Deplacer dans package\prog\archive\ ou package\manuels\archive\ — remplace.py ignorera ces dossiers automatiquement.

5. Deploiement multi-postes

  • Meme structure C:\SiteGITHUB\Hebreu4.0\ sur chaque machine
  • Autoriser git-credential-manager.exe dans le pare-feu sur chaque poste
  • Token GitHub propre a chaque utilisateur dans son config.yaml (ou laisser vide pour utiliser GCM)
  • Partager le ZIP package pour synchroniser les versions
