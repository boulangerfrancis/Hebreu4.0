1. Flux de travail quotidien

EtapeAction
1. Modifier le contenuEditer ou ajouter des .docx dans votre dossier documents\
2. Generer et verifierDouble-clic sur lancer.cmd — le site s'ouvre sur http://localhost:3500
3. PublierDouble-clic sur MAJ_GITHUB.cmd — publication sur GitHub Pages en 1-2 min

2. Organisation des documents sources

documents\
├── Grammaire\
│   ├── Lecon01.docx          → lien dans la rubrique Grammaire
│   └── Lecon02.docx
├── Vocabulaire\
│   └── Mots_courants.docx
└── Chants\
    ├── __partition_Ave.docx  ← prefixe __ = partition musicale
    └── __correspondance.csv  ← associe partition + lien YouTube

Regles de nommage

ConventionDescription
Nom libre.docxDocument normal — apparait comme lien dans le menu
__partition_NOM.docxPartition musicale — genere un PDF avec boutons YouTube
__correspondance.csvAssociation partition ↔ lien YouTube (meme dossier)

3. Generer le site (lancer.cmd)

  1. Double-clic sur lancer.cmd a la racine du projet
  2. Attendre la fin — le navigateur s'ouvre sur http://localhost:3500
  3. Verifier le resultat
lancer.cmd supprime html\ avant chaque generation pour eviter les fichiers residuels (anciens PDF, HTML orphelins).
La table des matieres est generee automatiquement en phase 5 dans html\tdm\index.html.

4. Publier sur GitHub Pages (MAJ_GITHUB.cmd)

  1. Double-clic sur MAJ_GITHUB.cmd
  2. La fenetre affiche 4 etapes : sync → generation → commit → push
  3. Confirmer si demande (confirmation_git = true dans config.yaml)
  4. Le site est en ligne sur https://guytitt.github.io/Hebreu4.0/
Authentification : si github_token est renseigne dans prog\config.yaml, le push est silencieux. Sinon, GCM (GitHub Desktop) prend la main — requiert que git-credential-manager.exe soit autorise dans le pare-feu Windows.

5. Consulter la documentation

  1. Double-clic sur manuels\serveur_manuels.cmd
  2. Le navigateur s'ouvre sur http://localhost:3501
  3. Ou ouvrir directement manuels\index.html

6. Ajouter ou modifier du contenu

Ajouter un document

  1. Copier le .docx dans le dossier documents\ voulu
  2. Lancer lancer.cmd — le document apparait automatiquement
  3. Verifier, puis MAJ_GITHUB.cmd

Supprimer un document

  1. Supprimer le .docx (et le .pdf correspondant)
  2. Relancer lancer.cmd puis MAJ_GITHUB.cmd

7. Partitions musicales

  1. Nommer le fichier : __partition_NomDeLaPartition.docx
  2. Creer __correspondance.csv dans le meme dossier :
pdf_name,youtube_url
nom_de_la_partition.pdf,https://youtube.com/watch?v=XXXX

Lancer lancer.cmd — un PDF avec boutons YouTube et Jouer ici est genere automatiquement.

8. Si quelque chose ne fonctionne pas

SymptomeSolution rapide
Le site ne se genere paspython prog\versions.py pour diagnostiquer
PDF non genereVerifier que PDFCreator est installe et configure
Table des matieres absente sur GitHubVerifier que le dossier s'appelle tdm (minuscules) et non TDM
Push GitHub echoue (403)Autoriser git-credential-manager.exe dans le pare-feu Windows
Push echoue (token 401)Regenerer un token PAT sur GitHub et le mettre dans config.yaml
Modules manquantspython package\prog\installer.py puis python package\prog\remplace.py
